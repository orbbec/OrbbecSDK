详细描述

该文件包含多个设备相关的 API 开关，用于控制多个设备之间的同步以及单个设备内不同传感器之间的同步

多个设备之间的同步是复杂的，不同的型号有不同的同步模式和限制。请参照有关详细信息，请参阅产品手册。 由于深度相机和红外相机在物理上是相同的传感器，因此红外相机的行为与同步模式中的深度相机相同

在文件 MultipleDevices.h 中定义.

void ob_device_trigger_capture ( ob_device *  device,
ob_error **  error 
)

发送捕获信息至设备

当设备处于 OB_MULTI_DEVICE_SYNC_MODE_SOFTWARE_TRIGGERING 中时，收到捕获命令后，设备将开始一次图像捕获

注意
用户调用此函数的频率乘以每个触发器的帧数应该小于流的帧速率。每个触发器的帧数可以由 framesPerTrigger 设置
对于某些型号，接收和执行捕获命令会有一定的延迟和性能消耗，因此调用该功能的频率不应过高，具体支持的频率请参阅产品手册
如果设备未处于 OB_MULTI_DEVICE_SYNC_MODE_HARDWARE_TRIGGERING 模式，则设备将忽略捕获命令

void ob_device_timestamp_reset ( ob_device *  device,
ob_error **  error 
)

发送时间戳重置命令至设备

当时间戳重置功能启用时，设备在接收到时间戳重置命令后，将用于计算输出帧的时间戳的计时器重置为 0。时间戳重置功能可以通过调用 ob_device_set_timestamp_reset_config 来启用。

注意
如果设备的流被启动，则流输出的连续帧的时间戳将在时间戳重置后跳一次
由于设备的定时器精度不高，流输出的连续帧的时间戳在长时间后会漂移。用户可以定期调用此功能来重置计时器，以避免时间戳漂移，建议间隔时间为 60 分钟。

)

将设备的计时器与主机同步

调用此功能后，设备的计时器将与主机同步。用户可以将此功能调用到多个设备，以同步设备的所有计时器

注意
如果设备的流被启动，则该流输出的连续帧的时间戳可能在定时器同步之后跳一次
由于设备的定时器精度不高，所以流输出的连续帧的时间戳在长时间后会出现漂移。用户可以通过定期调用此功能来同步计时器，以避免出现时间戳漂移现象，建议调用间隔时间为 60 分钟
